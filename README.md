# reportilo (browser app)

This is the `webapp` branch: a client-side React + TypeScript app that lets you
fill in EQUATOR reporting checklists and flow diagrams and export them to Word,
Excel or images, entirely in your browser. It is the front end companion to the
[reportilo R package](https://github.com/choxos/reportilo) and is published at
**<https://choxos.github.io/reportilo/app/>**.

The R package, its bundled Shiny app and this browser app all read the same data
(exported as JSON from the package), so results match across all three.

## Stack

- React 18 + TypeScript, built with Vite
- Tailwind CSS
- Graphviz in the browser via `@hpcc-js/wasm-graphviz` (flow diagram rendering)
- `docx` (Word), `write-excel-file` (Excel), canvas (PNG/SVG) for client-side export
- `dompurify` to sanitize generated SVG before injection

## Develop

```bash
npm install
npm run dev                              # http://localhost:5173
npm run typecheck
npm test                                 # Vitest unit tests
npm audit --omit=dev --audit-level=high  # production dependency gate (must be clean)
npm run build                            # outputs to dist/
```

The full `npm audit` (including dev dependencies) currently reports advisories in
the Vite/Vitest/esbuild dev toolchain. These do not affect the shipped static
site (the production audit above is clean); track the Vite/Vitest upgrade path
and re-test when fixed versions are available.

## Data

`public/data/*.json` is generated by the R package's data pipeline
(`data-raw/10_export_json.R`) and copied here. To refresh it, regenerate the
package data on `main` and copy `inst/extdata/*.json` into `public/data/`.

## Deploy

`.github/workflows/deploy-app.yaml` builds on every push to `webapp` and
publishes `dist/` into the `gh-pages` `app/` subfolder, alongside the pkgdown
site at the root.
